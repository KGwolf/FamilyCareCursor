# 微信订阅消息续授权策略

本方案用于处理每日/每周循环提醒的微信订阅消息授权问题。

## 核心思路

微信订阅消息不能作为永久无限次推送通道。对于循环提醒，用户每次打开小程序时，如果存在启用中的循环提醒，系统会用业务弹窗解释原因，并引导用户重新授权下一次提醒。

流程：

```text
用户打开首页
↓
检查是否存在 daily / weekly / custom_weekly 循环提醒
↓
检查今天是否已经提示过
↓
先弹业务说明弹窗
↓
用户点击“继续接收”
↓
调用 wx.requestSubscribeMessage
↓
记录授权结果到 notification_bindings
```

## 触发规则

当前实现满足以下条件才会提示：

- 已配置微信订阅模板 ID；
- 推送总开关未关闭；
- 当前通道是 `wechat_subscribe`；
- 存在启用中的循环提醒；
- 当天提示次数未超过设置值。

默认每天最多提示 2 次，可在设置页调整；设置为 0 表示不自动提示。目前不限制最大值，但不建议设置过大，避免打扰用户。

## 不解决的问题

这个方案不是永久推送。用户如果长期不打开小程序，就无法续订新的微信订阅授权。

因此它适合作为默认主链路：

```text
微信订阅消息 + 用户打开小程序时温和续授权
```

企业微信内部成员通道可以作为愿意加入企业内部成员体系的高级通道，不建议默认开启。
